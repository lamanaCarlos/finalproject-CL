# API REST – Documentación de Endpoints

Este documento define los contratos de la API REST que utilizará la plataforma de comercialización de arte. Sirve como referencia para el desarrollo coordinado de frontend y backend.

---

## 1. Convenciones Generales

- Base URL: `/api`
- Autenticación: JWT vía header `Authorization: Bearer <token>`
- Respuestas en formato JSON
- Control de acceso mediante middleware por rol

Roles:
- `buyer`
- `artist`
- `admin`

---

## 2. Autenticación

### 2.1 Registro de Usuario

**POST** `/auth/register`

Roles permitidos: público

Request:
```json
{
  "email": "user@email.com",
  "password": "string",
  "role": "buyer | artist"
}
```

Response:
```json
{
  "message": "User registered successfully"
}
```

---

### 2.2 Login

**POST** `/auth/login`

Request:
```json
{
  "email": "user@email.com",
  "password": "string"
}
```

Response:
```json
{
  "token": "jwt-token",
  "role": "buyer | artist | admin"
}
```

---

## 3. Usuarios

### 3.1 Obtener Perfil Propio

**GET** `/users/me`

Roles: buyer, artist, admin

Response:
```json
{
  "email": "user@email.com",
  "role": "artist"
}
```

---

## 4. Artistas

### 4.1 Crear / Actualizar Perfil de Artista

**POST** `/artists/profile`

Roles: artist

Request:
```json
{
  "displayName": "Nombre artístico",
  "bio": "Biografía",
  "socialLinks": {
    "instagram": "url",
    "web": "url"
  }
}
```

---

### 4.2 Obtener Perfil Público de Artista

**GET** `/artists/:id`

Roles: público

---

## 5. Obras

### 5.1 Crear Obra

**POST** `/artworks`

Roles: artist

Request:
```json
{
  "title": "Título",
  "description": "Descripción",
  "type": "digital | physical",
  "price": 100
}
```

---

### 5.2 Listado Público de Obras

**GET** `/artworks`

Roles: público

Query params:
- artistId
- minPrice
- maxPrice

---

### 5.3 Detalle de Obra

**GET** `/artworks/:id`

Roles: público

---

## 6. Pedidos / Ventas

### 6.1 Comprar Obra

**POST** `/orders`

Roles: buyer

Request:
```json
{
  "artworkId": "ObjectId"
}
```

---

### 6.2 Ver Pedidos del Usuario

**GET** `/orders/my`

Roles: buyer, artist

---

## 7. Encargos Personalizados

### 7.1 Solicitar Encargo

**POST** `/commissions`

Roles: buyer

Request:
```json
{
  "artistId": "ObjectId",
  "description": "Descripción del encargo",
  "budget": 300
}
```

---

### 7.2 Gestionar Encargo

**PATCH** `/commissions/:id`

Roles: artist

Request:
```json
{
  "status": "accepted | rejected | in_progress | completed"
}
```

---

## 8. Administración

### 8.1 Gestión de Usuarios

**GET** `/admin/users`

Roles: admin

---

### 8.2 Configurar Comisión

**PATCH** `/admin/settings`

Roles: admin

Request:
```json
{
  "minimumCommission": 10
}
```

---

## 9. Errores Comunes

- 401 Unauthorized
- 403 Forbidden
- 404 Not Found
- 500 Internal Server Error
