# Backlog del Producto – Marketplace de Arte

Este documento define el **backlog del producto** organizado en **épicas** e **historias de usuario**, siguiendo un enfoque Scrum. Sirve como base para planificación de sprints y control de alcance.

---

## Convenciones

Formato de historia:
> Como **[rol]** quiero **[objetivo]** para **[beneficio]**

Prioridad:
- **MVP**: imprescindible para lanzamiento
- **POST-MVP**: evolución

---

## ÉPICA 1 – Autenticación y Usuarios

### US-1 Registro de usuarios
**Historia**: Como visitante quiero registrarme como comprador o artista para acceder a la plataforma.

**Prioridad**: MVP

**Criterios de aceptación**:
- Registro con email y contraseña
- Selección de rol buyer o artist
- Validación de email único

---

### US-2 Login de usuarios
Como usuario quiero iniciar sesión para acceder a mis funcionalidades.

**Prioridad**: MVP

**Criterios de aceptación**:
- Login con credenciales válidas
- Recepción de JWT
- Redirección según rol

---

## ÉPICA 2 – Perfiles de Artista

### US-3 Crear perfil de artista
Como artista quiero crear mi perfil público para mostrar mi identidad artística.

**Prioridad**: MVP

**Criterios de aceptación**:
- Nombre artístico
- Biografía
- Imagen de perfil

---

### US-4 Editar perfil de artista
Como artista quiero editar mi perfil para mantenerlo actualizado.

**Prioridad**: MVP

---

## ÉPICA 3 – Gestión de Obras

### US-5 Crear obra
Como artista quiero crear una obra para venderla en la plataforma.

**Prioridad**: MVP

**Criterios de aceptación**:
- Selección de tipo (digital / física)
- Precio obligatorio
- Subida de imágenes

---

### US-6 Publicar / despublicar obra
Como artista quiero controlar la visibilidad de mis obras.

**Prioridad**: MVP

---

### US-7 Ver galería pública
Como visitante quiero ver obras públicas para descubrir artistas.

**Prioridad**: MVP

---

## ÉPICA 4 – Compra de Obras

### US-8 Comprar obra
Como comprador quiero comprar una obra para adquirirla legalmente.

**Prioridad**: MVP

**Criterios de aceptación**:
- Solo usuarios logueados
- Obra pasa a estado vendida

---

### US-9 Gestión de envío
Como artista quiero acordar el envío con el comprador para entregar la obra física.

**Prioridad**: POST-MVP

---

## ÉPICA 5 – Encargos Personalizados

### US-10 Solicitar encargo
Como comprador quiero solicitar un encargo para obtener una obra personalizada.

**Prioridad**: POST-MVP

---

### US-11 Gestionar encargo
Como artista quiero aceptar o rechazar encargos para controlar mi carga de trabajo.

**Prioridad**: POST-MVP

---

## ÉPICA 6 – Administración

### US-12 Aprobar artistas
Como administrador quiero aprobar artistas para garantizar calidad.

**Prioridad**: MVP

---

### US-13 Configurar comisión
Como administrador quiero definir la comisión mínima para controlar ingresos.

**Prioridad**: MVP

---

## ÉPICA 7 – Internacionalización

### US-14 Cambiar idioma
Como usuario quiero cambiar el idioma para usar la plataforma cómodamente.

**Prioridad**: MVP

---

## Backlog Vivo

Este backlog está sujeto a refinamiento continuo conforme avance el proyecto y se validen hipótesis de producto.
