# Plan de Despliegue y Entornos – Marketplace de Arte

Este documento define la estrategia de **despliegue, entornos y operación** del proyecto, asegurando estabilidad, seguridad y escalabilidad desde el inicio.

---

## 1. Entornos del Proyecto

Se definen tres entornos claramente separados:

### 1.1 Desarrollo (Development)
- Uso local por desarrolladores
- Hot reload y logs detallados
- Datos de prueba

### 1.2 Preproducción (Staging)
- Réplica de producción
- Validación funcional y QA
- Pruebas de integración

### 1.3 Producción (Production)
- Usuarios reales
- Alta disponibilidad
- Monitorización activa

---

## 2. Infraestructura Recomendada

### Frontend
- Hosting: Vercel / Netlify
- Build automático desde rama main
- HTTPS por defecto

### Backend
- Hosting: Render / Railway
- Node.js + Express
- Escalado horizontal

### Base de Datos
- MongoDB Atlas
- Backups automáticos
- Acceso restringido por IP

---

## 3. Variables de Entorno

Ejemplo:

```
NODE_ENV=production
PORT=4000
MONGO_URI=
JWT_SECRET=
CLOUDINARY_API_KEY=
STRIPE_SECRET_KEY=
```

Nunca se suben variables reales al repositorio.

---

## 4. CI / CD

Pipeline recomendado:
1. Push a rama main
2. Ejecución de tests
3. Build frontend
4. Deploy automático

Beneficios:
- Menos errores humanos
- Despliegues frecuentes y seguros

---

## 5. Seguridad

- HTTPS obligatorio
- JWT con expiración
- Rate limiting en API
- Protección contra inyecciones

---

## 6. Monitorización y Logs

- Logs centralizados
- Alertas por error
- Métricas básicas:
  - Uso
  - Errores
  - Latencia

---

## 7. Backups y Recuperación

- Backups diarios de base de datos
- Plan de restauración probado

---

## 8. Escalabilidad

- Separación frontend/backend
- Preparado para microservicios
- CDN para imágenes

---

## 9. Checklist Pre-Lanzamiento

- Variables de entorno configuradas
- Tests críticos superados
- Textos legales publicados
- Copias de seguridad activas

---

Documento final de la fase de preparación técnica para lanzamiento.
